BLUESTOCK MF DASHBOARD - POWER BI DEVELOPMENT GUIDE
===================================================

PREREQUISITE
------------
1. Install Power BI Desktop from Microsoft Store or microsoft.com/power-bi
2. Make sure all cleaned CSV files are in: C:\bluestock-mf-capstone\data\processed\
3. SQLite database ready at: C:\bluestock-mf-capstone\bluestock_mf.db


STEP 1: CONNECT DATA TO POWER BI
---------------------------------

Option A: Import CSV Files (Recommended - Easier)
1. Open Power BI Desktop
2. Click "Get Data" > "Text/CSV"
3. Import these 10 files one by one:
   - 01_fund_master_clean.csv
   - 02_nav_history_clean.csv
   - 03_aum_by_fund_house_clean.csv
   - 04_monthly_sip_inflows_clean.csv
   - 05_category_inflows_clean.csv
   - 06_industry_folio_count_clean.csv
   - 07_scheme_performance_clean.csv
   - 08_investor_transactions_clean.csv
   - 09_portfolio_holdings_clean.csv
   - 10_benchmark_indices_clean.csv

4. After importing, click "Transform Data" to check data types
5. Make sure date columns are formatted as Date type
6. Make sure amfi_code is Text type (not number)
7. Click "Close & Apply"

Option B: SQLite Connection (Advanced)
1. Download SQLite ODBC driver from: http://www.ch-werner.de/sqliteodbc/
2. Install the driver
3. In Power BI, click "Get Data" > "ODBC"
4. Configure DSN or use connection string:
   Driver=SQLite3 ODBC Driver;Database=C:\bluestock-mf-capstone\bluestock_mf.db
5. Select all tables and load


STEP 2: CREATE RELATIONSHIPS
-----------------------------
1. Click "Model" view (left sidebar icon)
2. Create these relationships by dragging fields:

   fund_master[amfi_code] → nav_history[amfi_code] (Many-to-One)
   fund_master[amfi_code] → scheme_performance[amfi_code] (Many-to-One)
   fund_master[amfi_code] → investor_transactions[amfi_code] (Many-to-One)
   fund_master[amfi_code] → portfolio_holdings[amfi_code] (Many-to-One)
   
   nav_history[date] → benchmark_indices[date] (Many-to-One)

3. Verify all relationships show as "1" to "*" (one-to-many)


STEP 3: CREATE MEASURES (NEW MEASURES)
---------------------------------------
Click "Modeling" tab > "New Measure" and create these DAX measures:

Total AUM = SUM(aum_by_fund_house[aum_cr])
Total SIP Inflows = SUM(monthly_sip_inflows[sip_inflow_cr])
Total Folios = MAX(industry_folio_count[total_folios_cr])
Total Schemes = DISTINCTCOUNT(fund_master[amfi_code])
Avg Return = AVERAGE(scheme_performance[return_3yr])
Avg StdDev = AVERAGE(scheme_performance[volatility_3yr])


STEP 4: PAGE 1 - INDUSTRY OVERVIEW
-----------------------------------
1. Insert New Page, rename to "Industry Overview"

2. Add 4 KPI Cards at top:
   Visual: "Card"
   - Card 1: Total AUM (format as ₹81L Cr)
   - Card 2: SIP Inflows (format as ₹31K Cr)
   - Card 3: Total Folios (format as 26.12 Cr)
   - Card 4: Total Schemes (format as 1,908)

3. Line Chart - AUM Trend:
   Visual: "Line Chart"
   X-axis: aum_by_fund_house[date] (or year)
   Y-axis: Total AUM
   Title: "Industry AUM Growth 2022-2025"

4. Bar Chart - AUM by AMC:
   Visual: "Clustered Bar Chart"
   Y-axis: aum_by_fund_house[fund_house]
   X-axis: aum_by_fund_house[aum_cr]
   Title: "AUM by Asset Management Company"
   Sort: Descending by AUM


STEP 5: PAGE 2 - FUND PERFORMANCE
----------------------------------
1. Insert New Page, rename to "Fund Performance"

2. Scatter Plot - Risk vs Return:
   Visual: "Scatter Chart"
   X-axis: scheme_performance[return_3yr]
   Y-axis: scheme_performance[volatility_3yr]
   Size: scheme_performance[aum_cr] (if available, else use fixed size)
   Details: fund_master[scheme_name]
   Title: "Risk-Return Profile"

3. Fund Scorecard Table:
   Visual: "Table"
   Columns:
   - fund_master[scheme_name]
   - scheme_performance[return_1yr]
   - scheme_performance[return_3yr]
   - scheme_performance[sharpe_ratio]
   - fund_master[expense_ratio_pct]
   Enable: Column sorting

4. Line Chart - NAV vs Benchmark:
   Visual: "Line Chart"
   X-axis: nav_history[date]
   Y-axis: nav_history[nav]
   Legend: Add benchmark_indices[index_name]
   Secondary Y-axis: benchmark_indices[close_value]

5. Add Slicers:
   Visual: "Slicer"
   - Slicer 1: fund_master[fund_house]
   - Slicer 2: fund_master[category]
   - Slicer 3: fund_master[plan]


STEP 6: PAGE 3 - INVESTOR ANALYTICS
------------------------------------
1. Insert New Page, rename to "Investor Analytics"

2. Bar Chart - Transactions by State:
   Visual: "Clustered Bar Chart"
   Y-axis: investor_transactions[state]
   X-axis: investor_transactions[amount]
   Title: "Investment by State"
   Sort: Top 10 states

3. Donut Chart - Transaction Type Split:
   Visual: "Donut Chart"
   Legend: investor_transactions[transaction_type]
   Values: COUNT or SUM(amount)
   Title: "SIP vs Lumpsum vs Redemption"

4. Bar Chart - Age Group vs Avg SIP:
   Visual: "Clustered Column Chart"
   X-axis: investor_transactions[age_group]
   Y-axis: AVERAGE(investor_transactions[amount])
   Title: "Average SIP Amount by Age Group"

5. Line Chart - Monthly Transaction Volume:
   Visual: "Line Chart"
   X-axis: investor_transactions[date] (by Month)
   Y-axis: COUNT(investor_transactions[transaction_id])
   Title: "Monthly Transaction Volume"

6. Add Slicers:
   - Slicer 1: investor_transactions[state]
   - Slicer 2: investor_transactions[age_group]
   - Slicer 3: investor_transactions[city_tier]


STEP 7: PAGE 4 - SIP & MARKET TRENDS
-------------------------------------
1. Insert New Page, rename to "SIP & Market Trends"

2. Dual-Axis Chart - SIP vs Nifty:
   Visual: "Line and Clustered Column Chart"
   X-axis: monthly_sip_inflows[month]
   Column Y-axis: monthly_sip_inflows[sip_inflow_cr]
   Line Y-axis: benchmark_indices[close_value] (filter for NIFTY50)
   Title: "SIP Inflows vs Market Performance"

3. Heatmap - Category Inflows:
   Visual: "Matrix"
   Rows: category_inflows[category]
   Columns: category_inflows[month]
   Values: category_inflows[net_inflow_cr]
   Enable: Conditional Formatting (Color Scale)

4. Bar Chart - Top 5 Categories FY25:
   Visual: "Clustered Bar Chart"
   Y-axis: category_inflows[category]
   X-axis: category_inflows[net_inflow_cr]
   Filter: FY25 (Apr 2024 - Mar 2025)
   Sort: Top 5


STEP 8: ADD INTERACTIVITY
--------------------------
1. Drill-Through Page:
   - Create new page "NAV Detail"
   - Add detailed NAV line chart
   - Right-click on fund table > "Drill through" > Link to NAV Detail page

2. Tooltips:
   - Select each visual > Format pane > Tooltips > Enable
   - Add relevant fields to tooltip

3. Cross-Filtering:
   - Select visual > Format pane > Edit Interactions
   - Set how other visuals respond when you click


STEP 9: APPLY THEME AND BRANDING
---------------------------------
1. View tab > Themes > Select professional theme (or customize)

2. Bluestock Color Scheme (use these hex codes):
   Primary: #003366 (Dark Blue)
   Secondary: #0066CC (Blue)
   Accent: #FF9900 (Orange)
   Background: #F5F5F5 (Light Gray)

3. Add Logo (if you have one):
   Insert > Image > Place Bluestock logo in top-right corner
   Resize to ~100x50 pixels

4. Consistent Formatting:
   - All titles: 14pt Bold
   - All axis labels: 10pt
   - Background: Light color
   - Grid: Subtle


STEP 10: EXPORT DELIVERABLES
-----------------------------
1. Save PBIX File:
   File > Save As > "bluestock_mf_dashboard.pbix"
   Location: C:\bluestock-mf-capstone\dashboard\

2. Export to PDF:
   File > Export > Export to PDF
   Save as: "Dashboard.pdf"
   Include: All pages

3. Export Page Screenshots:
   For each page:
   - View the page in full screen
   - File > Export > Export to Image (PNG)
   - Save as:
     * Page1_Industry_Overview.png
     * Page2_Fund_Performance.png
     * Page3_Investor_Analytics.png
     * Page4_SIP_Market_Trends.png


COMMON ISSUES AND FIXES
------------------------
Issue 1: Relationships not auto-detected
Fix: Manually create in Model view, ensure data types match

Issue 2: Date fields showing as text
Fix: Transform Data > Change Type > Date

Issue 3: Measures not calculating
Fix: Check table names match exactly, use fully qualified names

Issue 4: Charts showing too much data
Fix: Add Top N filter or date range slicer

Issue 5: SQLite ODBC not working
Fix: Use CSV import method instead (easier and more reliable)
